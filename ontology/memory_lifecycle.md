# Memory Lifecycle v0.2

Living memory moves through explicit states. No state silently erases history.

## observed

A signal, conversation, event, or condition is noticed. It is not yet stable truth.

## recorded

The observation is appended as a local memory event.

## classified

The event receives a memory kind, importance, confidence, source, and evidence.

## linked

The event is connected to other events through typed memory links.

## reviewed

The guardian or a validation process reviews the event, link, or derived view.

## reinforced

The event is strengthened through recall, repeated confirmation, or successful use.

## consolidated

The event contributes to a knowledge capsule, project state, preference, or snapshot.

## promoted

The memory becomes trusted within a defined scope. Promotion requires evidence and may require guardian confirmation.

## corrected

A later event corrects or supersedes an older interpretation. The old event remains visible.

## quarantined

The event is visible to audit but not trusted as operating context.

## archived

The event remains in history but is removed from normal active retrieval.

## Ordering rules

- Guardian-confirmed memories outrank unconfirmed memories.
- Correction memories outrank older conflicting memories within their scope.
- Quarantined memory must not be used as trusted context.
- Snapshots are derived views, not source of truth.
- Memory is not silently deleted.
