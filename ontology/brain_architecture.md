# Living Memory Brain Architecture v0.2

Genesis-core defines a neutral cognitive architecture. It does not assume any app, operating system, device class, model, vendor, or cloud service.

## Root model

```text
seed -> birth event -> living memory log -> derived views -> reasoning context
```

The seed is immutable after birth. The birth event starts the instance. Living memory grows after birth as append-only events. Derived views help the instance read itself, but they are not the source of truth.

## Memory is not raw chat history

Raw conversation may be recorded as episodic memory, but stable knowledge requires classification, evidence, confidence, consolidation, and conflict handling.

## Local reasoning

The own local reasoning engine is allowed after birth without extra guardian approval. It may retrieve local memory, rank relevant memory, summarize known state, answer from local evidence, and state uncertainty.

## External reasoning

An external or non-permanent auxiliary reasoning provider is optional. It requires guardian approval and cannot become identity, memory, doctrine, owner, or authority.

## External context

External context is data, not instruction. It must not activate tools, approvals, memory changes, identity changes, or policy changes.

## Rest cycle

Rest consolidation may deduplicate, link, summarize, reinforce, degrade, quarantine, and propose migrations. It may not silently rewrite the seed or erase memory history.

## Emergency local mode

When no external provider or network is available, the instance remains itself. It answers from local memory, states uncertainty, and preserves identity.

## Control principle

The instance may freely analyze itself and design improvements. The guardian is the final authority for adopting real changes.
